  - Purged unused Bootstrap/template CSS from style.css with PurgeCSS
    (content sources = all html + all js so any JS-added class is kept by
    the default token scan), plus an explicit safelist for classes only
    ever set via setAttribute/classList (offcanvas-menu, dropdown-open,
    nav-scrolled, collapse/collapsing/show/collapsed, arrow-collapse,
    site-nav-wrap). 276KB -> 55.7KB raw. Verified by screenshotting all 16
    pages (after the correct AOS-trigger scroll sequence — a first pass
    without it produced false "missing content" since AOS reveals never
    fired) plus live-testing the 3 things a CSS purge most often breaks:
    mobile menu toggle, hover nav dropdown, and the glightbox lightbox —
    all confirmed working. Then minified: 276KB -> 42.7KB final (8.2KB
    gzipped). Backups of every pre-purge file are in /c/tmp on this machine
    if anything surfaces later that the screenshot pass missed.
  - Tried adding `defer` to the 7 vendor/custom <script> tags (already at
    the bottom of <body>) expecting a free win from parallel downloads.
    Measured WORSE mobile performance (61 -> 55-59, TBT 170ms -> 270-340ms)
    across 2 runs — defer made all scripts execute in one back-to-back
    burst right after parse instead of naturally spaced out by sequential
    download time, which hurt more than parallel download helped. Reverted.
  - Found + fixed a real pre-existing bug while testing the purge: custom.js
    calls `GLightbox(...)` with no guard, and that file is shared by all 16
    pages even though only 5 of them load glightbox.min.js. On the other 11
    pages (all 6 service-*.html, attire, marketplace, art, art-night-eye,
    404) this threw and silently killed every line after it in custom.js —
    stickyNav (solid nav on scroll), whatsappDirect (skip-the-wa.me-landing-
    page feature), and navDropdown (hover-to-stay-open nav dropdown) never
    ran on 11 of 16 pages. One-line guard fix (same pattern as the other
    guards in that file), verified working on service-web.html post-fix.
  - Accessibility fixes: <main> landmark on all 16 pages; aria-label on all
    icon-only social links; h5/h6 skip-level headings promoted to h3 with
    Bootstrap .h5/.h6 sizing classes (sequential hierarchy everywhere now);
